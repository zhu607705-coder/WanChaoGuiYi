# MVP Design

## 核心结论

国内版 Demo 先验证王朝周期闭环：扩张、治理、土地结构、法统、继承和帝皇机制互相影响。全球版只保留接口。

## 体验目标

玩家每回合都要权衡三个问题：

1. 是否继续扩张。
2. 新领土是否能够治理。
3. 当前帝皇死后国家是否还能延续。

## MVP 模块

| 模块 | 第一版要求 |
| --- | --- |
| 地图 | 史域九州融合节点地图，40 到 60 个区域 |
| 回合 | 半年一回合 |
| 资源 | 粮食、金钱、人口、兵力、合法性 |
| 风险 | 土地兼并、民变、财政崩溃、继承危机、地方割据 |
| 战争 | 相邻区域行军，自动战斗结算，战报 |
| 内政 | 征税、征兵、赈灾、清丈、招贤、改革 |
| 法统 | 国家合法性、地区法统接受度、新占领整合 |
| 继承 | 年龄、健康、继承人、继位合法性、危机事件 |
| 人才 | 名将、谋士、财政官、改革官 |
| AI | 不同帝皇有不同扩张和治理倾向 |

## 回合流程

1. 玩家处理事件。
2. 查看财政、粮食、民心、土地兼并和继承风险。
3. 选择政策。
4. 调动军队，进攻或防守。
5. AI 行动。
6. 结算战争、财政、人口、土地、派系满意度。
7. 触发人才、灾害、叛乱、继承和外交事件。
8. 进入下一回合。

## 初始帝皇机制

| 帝皇 | 独特机制 | 强项 | 代价 |
| --- | --- | --- | --- |
| 秦始皇 | 六合同轨 | 制度统一、中央集权、工程动员 | 民力透支、继承风险 |
| 刘邦 | 布衣共天下 | 招降、人才吸纳、地方妥协 | 功臣坐大、异姓王风险 |
| 汉武帝 | 外朝远征 | 强军、边疆扩张、财政动员 | 财政压力、民生下降 |
| 曹操 | 挟令重组 | 屯田、战乱恢复、人才网罗 | 正统争议、士族疑虑 |
| 李世民 | 贞观君臣 | 军事、纳谏、人才、边疆经营 | 继承阴影、宗室压力 |
| 赵匡胤 | 杯酒释兵权 | 文官治理、财政、低兵变 | 军事进攻受限 |
| 朱元璋 | 重造基层 | 户籍、基层控制、反腐、农业恢复 | 高压政治、藩王隐患 |
| 康熙 | 多元调和 | 多民族治理、边疆、士人安抚 | 大帝国治理成本高 |

## 胜利条件

- 统一九州：控制关键区域并维持最低合法性。
- 三代延续：完成至少三次平稳继承。
- 制度胜利：完成核心改革，合法性、财政、土地结构达到稳定阈值。

## 成功标准

1. 秦始皇、刘邦、朱元璋、赵匡胤玩起来有明显差异。
2. 土地兼并、财政、民变和继承危机会改变玩家策略。
3. 统一后还有治理压力。
